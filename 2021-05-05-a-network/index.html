Reflections - Wired networking

Petri Ojala, Apr 5, 2021, Wired Network

Networking – let’s think about boring cables first.

People tend to focus on wireless access but what one really should be doing is to make sure that if there are wires around, anything possible should be connected with wires.

In our current apartment we have ethernet wall plates here and there and it’s been relatively easy to wire the place but as this has been built around 15 years afgo, the cabling is probably Cat5 or Cat5e as it cannot handle anything above 1 Gbit/s that well.

I think it’s time to set the bar higher – if possible, have at least 2.5 Gbit/s around the new apartment.

There all ethernet wall plates around the new apartment but the details are still unclear. As it’s a few years old construction, the cables could well handle 2.5 or even 5 Gbit/s.

If 10 Gbit/s over copper works, that will be amazing and I will have trouble not to use it.. Some local links will be 10GE with SFP+ direct attach.

Switches

The current dumb gigabit-switches and a few Mikrotik’s should be a good starting point. The Mikrotik’s have some SFP+ ports for faster connections and I have a few DA’s around.

There are some inexpensive 2.5GE switches arriving to the market, for example 5- and 8-port devices from TRENDnet. These should be fine to use so I might update some core switching devices if the price is right.

VLANs

This is a tricky part.. I don’t run separate VLANs for different devices or device classes right now. It didn’t feel like worth the effort. However the current IPv4 ranges are split to fictitious /24 blocks based on the devices so switching to separate VLANs should be relatively easy. IPv6 is dynamic though.

I don’t want to go too complex with the VLANs but perhaps it’s time to have a few. Thanks to all the auto-discovery and IoT devices, one more than likely needs to configure mDNS repeater between the VLANs.
