Reflections - Edge firewall

Petri Ojala, Apr 5, 2021, Internet Network Firewall

Edge router and firewall.

Currently on the edge I run Ubiquiti’s EdgeRouter 4. It’s a small device with three ethernet ports and one SFP port running embedded Linux and Vyatta for routing features. It can hardware offload a lot of packet pushing so it’s pretty capable device – up to 4 Gbit/s and 320kpps for large packets, and up to 1.8 Gbit/s and 3400kpps for small packets.

But it hasn’t received that much love from Ubiquiti recently, software updates have been occasional not that I’ve missed much features anyway.

Is it time to upgrade? Maybe, or maybe not, I’d like to switch to OPNsense but I don’t feel like it’s absolutely necessary.

OPNsense would have quite a lot of new features, it’s more like a firewall with a router and OPNsense just recently announced some new devices that are ready for the job. DEC690 is based on AMD’s emdedded Ryzen and can do up to 3.3 Gbit/s firewall, 600 Mbit/s VPN and is fanless like the EdgeRouter.

I could enjoy having some visibility to who’s knocking on the door, have a HAProxy for inbound connectivity, and get some flow statistics about the traffic.

I also have another EdgeRouter X that is used for AWS, Azure and Google Cloud VPN’s. It’s a separate device and can be used the same way as with DNA, both DNA and Elisa provide up to five public IPs. The VPN’s are IPSec but perhaps the hyperscalers will include WireGuard support at some point.

I do have a bigger wish though; I would like to manage the router with Terraform or Ansible. There isn’t much for EdgeOS and Terraform, the OPNsense provider for Terraform doesn’t seem to be that actively maintained either. With Ansible might be a bit easier.
